## 模型 API

模型 API 為您提供一個統一的閘道，可存取 200 多個領先的 AI 模型。每個端點皆相容於一種生態系統標準——OpenAI、Claude 或 Gemini——因此您現有的 SDK 與工具皆可不做修改直接使用。

### 基礎網址

```text
https://api.dgrid.ai
```

### 身分驗證

所有模型 API 端點皆透過 Bearer Token 進行驗證：

```http
Authorization: Bearer <DGRID_API_KEY>
```

相容 Claude 的 `POST /v1/messages` 端點還需要額外提供：

```http
anthropic-version: 2023-06-01
```

請依照[金鑰建立指南](https://blog.dgrid.ai/posts/2026-01-04/)取得您的金鑰，或使用[管理 API 金鑰](/zh-Hant/management-api-keys)介面以程式化方式管理金鑰生命週期。

### 模型命名

`model` 參數使用 `provider/model-name` 格式——例如 `openai/gpt-4o`——此格式在所有 DGrid 支援的模型中皆一致。呼叫[列出模型](/zh-Hant/api-reference/models/list-models)以查詢您的金鑰可使用的模型 ID。

### 端點分類

| 分類 | 基礎路徑 | 最適合用於 |
| --- | --- | --- |
| 聊天 | `/v1/chat/completions`、`/v1/responses`、`/v1/messages`、`/v1/models/{model}:generateContent` | 對話應用程式、多模態助理、工具呼叫 |
| 文字補全 | `/v1/completions` | 純提示詞文字生成與舊版整合 |
| 嵌入向量 | `/v1/embeddings`、`/v1/engines/{engine}/embeddings` | 搜尋、聚類、檢索、排序 |
| 圖像 | `/v1/images/generations`、`/v1/images/edits` | 跨模型格式的圖像生成與編輯 |
| 音訊 | `/v1/audio/speech`、`/v1/audio/transcriptions`、`/v1/audio/translations` | 語音合成、轉錄、翻譯 |
| 內容審核 | `/v1/moderations` | 內容安全篩選與政策檢查 |
| 模型 | `/v1/models` | 查詢可用模型與支援的端點類型 |
| 即時對話 | `/v1/realtime`（WebSocket） | 低延遲串流對話 |

### 最佳實務

1. 將 DGrid API 金鑰儲存於伺服器端，切勿將其暴露於公開的客戶端或程式碼庫中。
2. 若希望取得最廣泛的 SDK 相容性並降低遷移成本，建議優先使用相容 OpenAI 的格式。
3. 僅在需要特定供應商的負載特性（例如 Gemini 多模態內容部分或 Claude 工具負載）時，才使用供應商原生格式。
4. 確保請求與回應範例與您所使用的目標模型系列保持一致，因為 OpenAI、Claude 與 Gemini 風格之間的參數名稱有所不同。

## 管理 API 金鑰

管理 API 金鑰是用於管理模型 API 金鑰完整生命週期的管理憑證——包括建立、輪換、使用限制、停用與撤銷。這類金鑰僅限於金鑰管理操作，無法呼叫模型 API 的推論端點。

請參閱[管理 API 金鑰](/zh-Hant/management-api-keys)，了解 API 介面、驗證邊界與端點參考文件。

## x402 API

x402 API 是 DGrid 的按次推論付費存取模式。無需建立帳戶或預先儲值，每次請求皆透過 x402 支付協定逐次授權並結算——非常適合自主代理與隨需整合的場景。

請參閱 [x402 API](/zh-Hant/x402/overview) 了解付款流程，以及 [x402 API 參考文件](/zh-Hant/x402/api-reference)了解端點詳情。
