## Model API

Model API는 200개 이상의 주요 AI 모델에 대한 단일 게이트웨이를 제공합니다. 모든 엔드포인트는 OpenAI, Claude, Gemini 등 생태계 표준과 호환되므로, 기존 SDK와 도구를 변경 없이 그대로 사용할 수 있습니다.

요청 단위 비용을 대조해야 한다면 `DGrid-Request-ID` 응답 헤더를 저장하고 [요청 결제 세부 정보 조회](/ko/api-reference/usage-and-billing/get-request-billing-details)를 사용하여 토큰 사용량, 가격, USD 비용 세부 정보를 확인하세요.

### Base URL

```text
https://api.dgrid.ai
```

### 인증

모든 Model API 엔드포인트는 베어러 토큰으로 인증합니다.

```http
Authorization: Bearer <DGRID_API_KEY>
```

Claude 호환 `POST /v1/messages` 엔드포인트는 추가로 다음을 요구합니다.

```http
anthropic-version: 2023-06-01
```

[키 생성 가이드](https://blog.dgrid.ai/posts/2026-01-04/)를 따라 API 키를 발급받거나, [Management API Keys](/ko/management-api-keys) 영역을 사용하여 프로그래밍 방식으로 키 생명주기를 관리할 수 있습니다.

### 모델 명명 규칙

`model` 매개변수는 `provider/model-name` 형식(예: `openai/gpt-4o`)을 사용하며, DGrid가 지원하는 모든 모델에서 동일하게 적용됩니다. 사용 가능한 모델 ID를 확인하려면 [모델 목록 조회](/ko/api-reference/models/list-models)를 호출하세요.

### 엔드포인트 패밀리

| 패밀리 | 기본 경로 | 적합한 용도 |
| --- | --- | --- |
| Chat | `/v1/chat/completions`, `/v1/responses`, `/v1/messages`, `/v1/models/{model}:generateContent` | 대화형 앱, 멀티모달 어시스턴트, 도구 호출 |
| Completions | `/v1/completions` | 프롬프트 전용 텍스트 생성 및 레거시 통합 |
| Embeddings | `/v1/embeddings`, `/v1/engines/{engine}/embeddings` | 검색, 클러스터링, 검색 증강 생성, 랭킹 |
| Images | `/v1/images/generations`, `/v1/images/edits` | 다양한 모델 형식의 이미지 생성 및 편집 |
| Audio | `/v1/audio/speech`, `/v1/audio/transcriptions`, `/v1/audio/translations` | 음성 합성, 음성 인식, 음성 번역 |
| Moderations | `/v1/moderations` | 콘텐츠 안전 필터링 및 정책 검사 |
| Models | `/v1/models` | 사용 가능한 모델 및 지원되는 엔드포인트 유형 조회 |
| Usage & Billing | `/api/v1/model-router/billing-json` | 요청 단위 토큰 사용량, 가격 스냅샷, USD 비용 세부 정보 조회 |
| Realtime | `/v1/realtime` (WebSocket) | 저지연 스트리밍 대화 |

### 모범 사례

1. DGrid API 키는 서버 측에 저장하고, 퍼블릭 클라이언트나 저장소에 절대 노출하지 마세요.
2. 가능한 최소한의 마이그레이션으로 가장 광범위한 SDK 호환성을 얻으려면 OpenAI 호환 형식을 사용하세요.
3. Gemini의 멀티모달 파트나 Claude의 도구 페이로드와 같은 제공자별 페이로드 기능이 필요한 경우에만 제공자 네이티브 형식을 사용하세요.
4. OpenAI, Claude, Gemini 스타일 간에 매개변수 이름이 다르므로, 요청 및 응답 예시를 대상 모델 패밀리에 맞게 일치시키세요.

## Management API Keys

Management API Keys는 Model API 키의 전체 생명주기 — 생성, 교체, 사용량 한도 설정, 비활성화, 폐기 — 를 위한 관리용 자격 증명입니다. 키 관리 작업으로만 제한되며 Model API 추론 엔드포인트를 호출할 수 없습니다.

API 영역, 인증 경계, 엔드포인트 참조에 대한 자세한 내용은 [Management API Keys](/ko/management-api-keys)를 참고하세요.

## x402 API

x402 API는 DGrid의 추론 단위 결제 접근 방식입니다. 계정을 생성하거나 잔액을 미리 충전하는 대신, x402 결제 프로토콜을 사용하여 호출별로 요청이 승인되고 정산됩니다 — 자율 에이전트 및 온디맨드 통합에 적합합니다.

결제 흐름은 [x402 API](/ko/x402/overview)를, 엔드포인트 세부 정보는 [x402 API Reference](/ko/x402/api-reference)를 참고하세요.
